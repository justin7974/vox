## 待讨论的 9 条功能（第 5-13 条）

请和 Justin 逐条讨论方案。每条需要确认：**做不做、怎么做、优先级**。

---

### 第 5 条：录音波形浮窗
**问题**：录音时用户不确定"系统在听吗"，黑箱感强。
**建议**：在现有 StatusOverlay 上加一个实时音频波形/电平条。技术上已有 `recorder.onAudioLevel` 回调，overlay 已有 `updateAudioLevel()` 方法。工作量小。
**讨论点**：波形样式（简单电平条 vs 波浪线 vs 频谱）、是否显示录音时长。

### 第 6 条：结果预览窗
**问题**：润色完直接粘贴，用户没机会审查。重要场景（发邮件）可能不放心。
**建议**：润色完弹一个小浮窗显示结果，用户可选「粘贴」「重录」「编辑后粘贴」。
**讨论点**：是否默认开启还是作为可选模式？预览窗的交互细节？会不会让流程变慢影响体验？

### 第 7 条：历史记录
**问题**：说过的话找不到，想重新复制某段之前的输入。
**建议**：菜单栏加一个最近 N 条语音输入的子菜单，点击可重新复制到剪贴板。
**讨论点**：保存多少条？是存原始文本还是润色后的？是否持久化到文件？

### 第 8 条：本地 Whisper 模式
**问题**：目前 ASR 必须联网，有隐私顾虑+网络依赖。
**建议**：集成 whisper.cpp，让 ASR 完全本地化。零成本、离线可用、隐私友好。
**讨论点**：工作量较大（需要编译 C 库 + Swift 桥接）。是否值得投入？本地 Whisper 的识别质量是否够用？macOS 自带的语音识别是否可以作为替代？

### 第 9 条：静音检测
**问题**：用户说完话后还要手动按热键停止。
**建议**：检测到连续 N 秒静音后自动停止录音。
**讨论点**：静音阈值和等待时间设多少合适？用户说话中间的正常停顿怎么处理？是否作为可选模式（有人喜欢手动控制）？

### 第 10 条：录音上限保护
**问题**：忘关录音 → 长时间录制 → 浪费 ASR API 额度。
**建议**：超过 N 分钟（比如 3 分钟）自动停止录音并处理。
**讨论点**：时间上限设多少？需要提示用户吗？工作量极小（几行代码）。

### 第 11 条：Prompt 模板市场
**问题**：用户需要自己写 prompt，门槛高。
**建议**：预设几套 prompt 模板（口语风、商务风、学术风、英文风），菜单里一键切换。
**讨论点**：和翻译模式是否冲突？模板放在哪里（内置 vs 文件）？需要多少模板够用？

### 第 12 条：快捷指令集成
**问题**：Vox 只能作为独立工具使用，无法嵌入更大的自动化流程。
**建议**：支持 macOS Shortcuts 触发录音/处理。
**讨论点**：是否有实际需求？实现需要 Intents 框架，工作量不小。优先级可能不高。

### 第 13 条：多热键绑定
**问题**：切换翻译模式、切换 prompt 模板都需要打开菜单。
**建议**：不同热键绑定不同模式。比如 `Cmd+Shift+1` 正常模式，`Cmd+Shift+2` 翻译模式。
**讨论点**：和第 11 条（模板）联动。Carbon API 支持注册多个热键。用户怎么配置这些热键？
